# 🌾 보리네 알리미

### 시니어 친화 일정(할 일) 알림 앱

### Expo + React Native / Android 중심

---

## 1. 앱 개요

보리네 알리미는 시니어가 **하루 할 일을 아침/점심/저녁 단위로 등록**하고,
정해둔 시간에 **미완료가 있을 때만 알림**을 받아 체크할 수 있는 일정 관리 앱이다.

로그인 없이 **기기 내부에만 저장**되며, 필요 시 **기록을 텍스트로 내보내기/공유**할 수 있다.

---

## 2. 해결하려는 핵심 문제

- “오늘 뭐 해야 하지?”를 매일 확인하기 어려움
- 해야 할 일이 있어도 시간이 지나서 잊어버림
- 목록 앱은 복잡해서 사용이 부담됨
- 기록을 공유/보관하고 싶지만 방법이 어려움

👉 해결 원칙:
**하루 흐름(아침/점심/저녁)에 맞춰 단순하게 등록하고, 체크하고, 필요하면 공유하기**

---

## 3. 핵심 UX 컨셉

- 하단 탭 4개로 단순한 이동(캘린더 / 오늘 할일 / 할일관리 / 설정)
- 오늘 할 일은 **오늘 날짜 + 요일**을 크게 표시
- 할 일은 **아침/점심/저녁** 버튼으로 바로 완료/미완료 체크
- 캘린더에서 날짜를 눌러 **그날 기록 요약 + 상세 목록** 확인
- 설정에서 알림 시간/알림 ON·OFF/기록 공유/개인정보처리방침 접근

---

## 4. 전체 UX 흐름

1. 할일 추가하기에서 할 일을 등록(기간/반복/아침·점심·저녁 선택)
2. 오늘 할일 화면에서 오늘 일정 확인
3. 아침/점심/저녁 버튼을 눌러 완료 체크(다시 누르면 미완료로 되돌림)
4. 캘린더에서 날짜 선택 → 해당 날짜의 완료/미완료를 확인
5. 설정에서 알림 시간을 조정하고, 기록공유하기로 월별 기록을 텍스트로 공유

---

## 5. 주요 기능 상세

### 5.1 할 일 등록

- 시작일/종료일: 캘린더로 선택
- 반복: 매일 / 매주(요일 선택)
- 시간대: 아침/점심/저녁 중 선택
- 같은 이름이 이미 있으면 자동으로 `이름(2)`처럼 번호를 붙여 중복 방지

---

### 5.2 오늘 할일 체크

- 오늘 날짜의 할 일만 보여줌
- 각 할 일에 대해 아침/점심/저녁 버튼으로 완료 체크
- 버튼은 위치가 고정되어(없는 시간대는 빈 칸) 한눈에 비교 가능

---

### 5.3 캘린더(기록 화면)

- 날짜를 누르면 해당 날짜의 기록을 아래에 리스트업
- 아침/점심/저녁 상태가 `완료/미완료`로 표시됨
- 날짜별로 “총 n개 중 m개 완료”를 확인 가능

---

### 5.4 할일 관리(수정/삭제)

- 등록된 할 일 목록을 확인
- 할 일 수정/삭제 가능
- 하단에 “할일 추가하기” 버튼 제공

---

### 5.5 알림(로컬 알림)

- 설정에서 아침/점심/저녁 알림 시간을 설정
- 알림 ON/OFF는 즉시 저장됨(별도 저장 버튼 없음)
- **해당 시간대에 미완료 할 일이 있을 때만** 알림이 오도록 동기화
- 알림 메시지에는 “지금 해야 할 일이 있어요”처럼 행동을 유도하는 문구를 포함

---

### 5.6 기록 공유(내보내기)

- 설정에서 월을 선택해 해당 월의 기록을 텍스트로 생성
- 생성된 텍스트는 OS 공유 기능으로 전송(메시지/메일 등)
- 기록이 없는 월은 선택 목록에서 제외(기록이 있는 월만 표시)

---

## 6. 광고 UX 정책

### 기본 원칙

- 기능 흐름을 방해하지 않음
- 자동 재생/소리/팝업 광고 ❌
- 화면 하단 배너로만 노출

### 참고

- 개발/테스트 환경에서는 `Test Ad`로 보일 수 있음

---

## 7. 개인정보/데이터 정책

- 로그인/회원가입 없음
- 할 일/기록/설정은 로컬 저장소에만 저장됨
- 개인정보처리방침은 앱 설정에서 링크로 제공됨

---

## 8. 기술 스택 및 구조

- Expo + React Native
- TypeScript
- 로컬 저장: AsyncStorage 기반
- 로컬 알림: `expo-notifications`
- 배너 광고: `react-native-google-mobile-ads`

---

## 9. 상태 관리 개념

- `tasks`: 등록된 할 일(기간/반복/시간대 포함)
- `history`: 날짜별 완료/미완료 기록
- `notificationEnabled`: 알림 ON/OFF
- `mealTimes`: 아침/점심/저녁 알림 시각

---

## 10. 제한 사항

- 기기 변경 시 로컬 데이터는 자동으로 옮겨지지 않음
- 알림 기능은 기기 설정(배터리 최적화/알림 권한)에 영향을 받을 수 있음
- 일부 기능은 개발 빌드(Dev Client)에서 테스트가 필요할 수 있음

---

## 11. 실행/빌드

루트에서 실행:

- 개발 서버: `npm run alimi:start`
- 안드로이드 실행: `npm run alimi:android`
